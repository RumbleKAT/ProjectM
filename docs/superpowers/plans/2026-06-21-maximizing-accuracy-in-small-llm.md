# 소형 파라미터 모델(SLM) RAG/대화 정확도 극대화 기술 플랜

## 1. 개요 (Summary)
로컬 환경에서 구동하는 소형 파라미터 모델(Small Language Models: Llama-3-8B, Mistral-7B, Qwen-2.5-7B 등)은 GPU 자원이 제한된 환경에서 경제적이지만, 대형 모델(GPT-4o, Claude 3.5 Sonnet 등)에 비해 추론 능력과 컨텍스트 이해도가 낮아 환각(Hallucination)이 잦고 프롬프트 지시사항을 무시하는 경향이 있습니다.

이 플랜은 ProjectM 환경에서 소형 모델을 사용할 때 RAG 및 대화 정확도를 상용 대형 LLM 수준으로 끌어올리기 위한 구체적인 시스템 설정 및 최적화 기법을 제시합니다.

---

## 2. 핵심 성능 저하 요인 및 대책 (Challenges & Mitigations)

| 요인 (Challenges) | 소형 모델의 증상 | 해결 대책 (Mitigations) |
| :--- | :--- | :--- |
| **맥락 소실 (Lost in the Middle)** | 컨텍스트가 너무 길면 본문 중간의 핵심 정보를 인지하지 못함 | **리랭커(Reranker) 도입** 및 **청크 컨텍스트 압축** |
| **지시문 불이행 (Instruction Following)** | 복잡한 제약 조건이나 JSON 출력 요구 포맷을 위반함 | **Few-Shot 예시 제공** 및 **시스템 프롬프트 간소화** |
| **낮은 추론 능력 (Reasoning)** | 멀티스텝 추론이나 복잡한 관계망 파악에 약함 | **단계별 추론(CoT)** 유도 및 **자기 반성(Self-Reflection) 루프** |
| **환각 현상 (Hallucination)** | 질문에 답하기 위해 본문에 없는 허구 사실을 지어냄 | **엄격한 그라운딩(Grounding) 룰** 강제 적용 |

---

## 3. 세부 최적화 플랜 (Optimization Details)

### ① RAG 파이프라인 고도화 (인풋 최적화)
소형 모델은 입력 데이터의 '노이즈'에 매우 취약합니다. 따라서 검색 엔진 단계에서 완벽히 필터링된 지식만 공급해야 합니다.

1.  **크로스 인코더 리랭커 (Reranker) 적용**:
    *   벡터 유사도 검색(Cosine Similarity)으로 뽑은 상위 20개 문서 조각을 **BGE-Reranker**나 **Cohere Reranker**를 통해 재정렬합니다.
    *   최종적으로 가장 관련성이 높은 최상위 3~5개 청크만 골라 소형 모델에 전달하여 입력 토큰 크기를 줄이고 정확도를 높입니다.
2.  **청크 분할 전략 최적화 (Chunk Overlap)**:
    *   문맥이 끊기지 않도록 문장 단위 경계를 유지하며, 청크 크기는 500~800 토큰 내외로 작게 유지하고 오버랩(Overlap) 비율을 20% 이상 확보합니다.

### ② 시스템 프롬프트 엔지니어링 (프롬프트 최적화)
소형 모델의 주의력(Attention)을 극대화하도록 정밀 프롬프트를 구성합니다.

1.  **Few-Shot 프롬프팅 제공**:
    *   질문과 본문, 그리고 바람직한 답변의 쌍(Pair) 예시를 2개 이상 프롬프트에 포함시킵니다. 소형 모델은 예시가 주어질 때 지시 이행 능력이 비약적으로 상승합니다.
2.  **명확하고 엄격한 제약사항 부여 (Grounding)**:
    *   `"너는 본문에 기재된 정보로만 답해야 한다. 본문에 답이 없으면 절대 추측하지 말고 '모르겠습니다'라고 답하라."`와 같은 거부 문구를 최상단과 최하단에 중복 배치합니다.
3.  **단계별 생각 유도 (Chain of Thought)**:
    *   `"답변하기 전에 본문에서 질문과 관련된 단서를 먼저 한 줄씩 찾아보고, 이를 바탕으로 최종 결론을 내리시오."`와 같이 추론 단계를 세분화합니다.

### ③ 추론 파라미터 조정 (하이퍼파라미터 최적화)
무작위성을 제어하여 팩트 기반 응답을 보장합니다.

*   **Temperature (온도)**: `0.0` ~ `0.2`로 매우 낮게 조정하여 매 질문마다 일관되고 정밀한 팩트 기반 답변을 출력하도록 강제합니다.
*   **Repetition Penalty (반복 패널티)**: `1.1` ~ `1.15` 수준으로 세팅하여 소형 모델 특유의 동일 단어/문장 무한 루프 반복 버그를 차단합니다.

### ④ 자기 반성(Self-Reflection) 검증 루프 적용
단일 추론의 한계를 보완하기 위해 시스템 내부에서 검증 패스를 한 번 더 돌립니다.

*   **2-Pass 검증 구조**:
    1.  소형 모델 A가 RAG 문서를 기반으로 초안 답변을 생성합니다.
    2.  소형 모델 A(또는 더 작은 초고속 모델 B)에게 `"초안 답변이 제공된 문서의 내용과 일치합니까? 예/아니오로 답하고 어긋난 부분이 있다면 수정본을 출력하시오."`라는 검증 프롬프트를 전송하여 자체 교정된 최종 응답을 유저에게 반환합니다.
