# 02. 초기 온보딩 및 모델/벡터 DB 설정

로컬 개발 서버를 실행한 후 브라우저로 처음 접속하면 시스템을 사용하기 전 필수 설정을 유도하는 온보딩(Onboarding) 화면이 나타납니다. 이 장에서는 서비스 이용을 위한 초기 설정 단계를 상세히 기술합니다.

---

## 1. 관리자 계정 생성 및 온보딩 시작
1.  브라우저를 열고 `http://localhost:5173`에 접속합니다.
2.  첫 환영 메시지와 함께 **"Get Started"** 버튼을 누릅니다.
3.  **기본 사용자 유형 설정**: single-user(1인 로컬 전용) 또는 multi-user(다중 사용자 로그인 및 관리자-회원 구조) 모드를 설정합니다.
4.  시스템 관리용 **로그인 계정 ID**와 **비밀번호**를 안전하게 설정하고 생성합니다.

---

## 2. LLM(대형 언어 모델) 제공업체 선택 및 연결
대화 응답 및 추론을 위한 핵심 두뇌가 될 LLM 연결을 설정합니다.

### A. OpenAI (상용 클라우드 모델)
*   **준비물**: OpenAI API Key
*   **설정**: LLM 목록에서 'OpenAI'를 고른 후 발급받은 API 키를 입력합니다. 대화 모델(`gpt-4o`, `gpt-3.5-turbo` 등)을 드롭다운에서 선택할 수 있습니다.

### B. Ollama (로컬 오픈소스 모델)
*   **준비물**: Ollama 프로그램 설치 및 모델 다운로드 (`ollama run llama3`)
*   **설정**: LLM 목록에서 'Ollama'를 선택합니다. Ollama가 기동 중인 로컬 URL(기본: `http://127.0.0.1:11434`)을 입력하고, 로컬 디바이스에 다운로드된 활성 모델명(예: `llama3`, `mistral`)을 입력한 뒤 연결을 테스트합니다.

---

## 3. 벡터 데이터베이스(Vector DB) 및 임베딩 설정
RAG 기술을 사용하려면 입력된 문서에서 유의미한 구절을 검색해야 하므로, 문장을 벡터화할 **임베딩 모델**과 가공된 벡터 데이터를 담을 **벡터 스토어(Vector DB)**가 필요합니다.

### A. 벡터 데이터베이스 연결 설정
*   **LanceDB (내장형 - 권장)**: 별도의 외부 데이터베이스 설치가 전혀 필요하지 않으며 서버 내부의 로컬 디스크 파일 시스템을 활용하므로 즉시 사용하기에 가장 적합하고 빠릅니다.
*   **Pinecone (클라우드 서비스)**: Pinecone API Key와 Index URL, 환경 리전 명칭을 입력하여 클라우드 벡터 스페이스를 연동합니다.
*   **PGVector (PostgreSQL 기반)**: PostgreSQL 인스턴스 연결 URL 및 벡터 저장용 스키마 명칭을 지정합니다.

### B. 임베딩 모델(Embedding Engine) 설정
*   텍스트 조각을 숫자 배열(Vector)로 바꿀 인코더 모델을 지정합니다.
*   OpenAI 임베딩 API(`text-embedding-3-small`) 또는 Ollama 로컬 임베딩, HuggingFace 내장 모델 중 하나를 연결합니다.

*설정이 완료되면 설정 테스트 후 메인 대시보드로 진입하여 가상 워크스페이스를 만들 준비가 완료됩니다.*
