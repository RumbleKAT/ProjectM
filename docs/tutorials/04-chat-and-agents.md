# 04. 채팅 스레드 관리, AI 에이전트 및 MCP 연동

이 장에서는 채팅 관리 요령과 음성 처리(TTS/STT), 자율 AI 에이전트 구축 및 외부 도구 연동 규격인 MCP(Model Context Protocol)의 연동 방법을 학습합니다.

---

## 1. 채팅 시스템 및 스레드(Thread) 관리
ProjectM의 대화 시스템은 단일 워크스페이스 내에서 여러 주제별로 대화를 쪼갤 수 있도록 **'스레드(Thread)'** 시스템을 지원합니다.

*   **스레드란?**: 동일한 문서 세트를 기반으로 하되, 각기 다른 연구 맥락이나 대화 주제를 별도로 유지하는 서브 대화방입니다.
*   **사용법**: 대화창 상단 혹은 좌측 하단의 **"New Thread"** 버튼을 클릭하여 새 스레드를 개설합니다. 이전 질문 및 답변 히스토리는 각 스레드 안에만 유지되므로, 컨텍스트 윈도우 한계를 낭비하지 않고 깨끗한 상태로 대화를 이어나갈 수 있습니다.

---

## 2. 멀티미디어 기능: 음성 입출력 (TTS / STT)
ProjectM은 텍스트를 소리로 들려주는 TTS 및 마이크 음성을 텍스트로 치환하는 STT 기능을 제공합니다.

### A. TTS (Text-to-Speech) 설정
AI가 타이핑하는 텍스트 답변을 음성으로 송출합니다.
1.  **설정(Settings) -> Audio Preference** 메뉴로 이동합니다.
2.  사용할 TTS 서비스 제공업체를 지정합니다:
    *   **Native Browser**: 운영체제 및 브라우저 기본 TTS 엔진을 사용합니다 (무료).
    *   **Piper**: 고품질 로컬 오픈소스 음성 합성 모듈을 활성화합니다.
    *   **ElevenLabs**: API 키를 입력하여 글로벌 최고 수준의 초자연적 음성을 연결합니다.

### B. STT (Speech-to-Text) 설정
대화창의 마이크 아이콘을 눌러 실시간 음성 인식을 통해 타이핑 없이 음성으로 질의할 수 있으며 브라우저 기본 Web Speech API를 사용하여 즉각 연동됩니다.

---

## 3. 자율 AI 에이전트 (AI Agents) 및 스킬(Skills)
워크스페이스 설정의 **"Agent Settings"** 탭을 통해 AI를 단순한 질의응답 비서에서 스스로 도구를 사용하는 '자율 에이전트'로 진화시킬 수 있습니다.

*   **에이전트 개념**: AI가 필요할 때 스스로 구글 검색을 하거나 파일을 읽고 쓰는 등의 외부 '도구'를 호출해 문제를 해결하도록 지시를 내리는 방식입니다.
*   **스킬 활성화**: 제공되는 에이전트 도구 중에서 Web Search(웹 브라우징), File Reader(파일 상세 판독), Gmail/Calendar 연동 등의 스킬들을 관리자 화면에서 활성화/비활성화 할 수 있습니다.

---

## 4. MCP (Model Context Protocol) 도구 연동
ProjectM은 Anthropic 사가 주도하는 표준 도구 연동 규격인 **Model Context Protocol (MCP)**을 완벽히 준수합니다.

### MCP 연동 방법
1.  대시보드 관리자 메뉴의 **"MCP Servers"** 설정으로 접속합니다.
2.  연결하려는 외부 MCP 서버의 API 주소 또는 로컬 Node/Python 실행 명령어 정보(예: MCP 데이터베이스 쿼리 도구, 로컬 파일 탐색 도구)를 작성하여 서버를 추가합니다.
3.  연동 성공 시 에이전트가 지원 규격을 감지하여 사용자가 질문한 내용에 따라 외부 MCP 서버의 도구를 실행하고 결합하여 최종 지식 응답을 구성합니다.
