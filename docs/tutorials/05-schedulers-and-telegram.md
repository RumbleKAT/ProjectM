# 05. 백그라운드 스케줄러 및 텔레그램 봇 연동

이 장에서는 백그라운드 관리 작업 스케줄러(System Jobs) 설정 방법과 텔레그램 메신저 앱을 대화형 인터페이스로 연동하는 방법을 최종적으로 알아봅니다.

---

## 1. 백그라운드 스케줄러 (Bree) 및 시스템 작업 (System Jobs)
ProjectM은 Node.js의 프로세스 격리 스케줄러 라이브러리인 `Bree`를 활용하여 백엔드 서버 부팅 시 백그라운드 청소 및 보수 작업을 자동으로 실행합니다.

관리자는 대시보드의 **"Scheduled Jobs" -> "System Jobs"** 탭에서 실행 상태를 제어할 수 있습니다.

### 핵심 시스템 작업 (System Jobs)
최근 추가된 자동 최적화 작업을 포함하여 크게 두 가지의 디폴트 시스템 작업이 백그라운드에서 동작합니다.

#### ① Cleanup inactive chat threads (오래된 대화방 자동 청소)
*   **설명**: 사용하지 않는 채널 리소스를 절약하기 위해 지정된 일수 동안 새로운 대화가 기록되지 않은 유휴 대화방(Thread)을 자동 삭제합니다.
*   **설정**: 시스템 작업 탭 상단의 카드에서 기간(`1일`, `1주일`, `1달`)을 선택할 수 있습니다.
*   **특징**: 시스템 안전을 위해 기본적으로 활성화(`enabledByDefault: true`)되어 기동됩니다.

#### ② Cleanup inactive workspaces (오래된 워크스페이스 자동 청소)
*   **설명**: 오랫동안 갱신되거나 채팅이 발생하지 않은 방치된 워크스페이스 전체(내부 문서, 대화 이력, 벡터 스토어 네임스페이스 포함)를 일괄 자동 삭제하여 디스크 및 클라우드 벡터 데이터베이스 비용을 절약합니다.
*   **설정**: 시스템 작업 탭 상단의 설정 카드에서 기간(`1일`, `1주일`, `1달`)을 구성합니다.
*   **특징**: 파괴적인 삭제 작업이므로 기본적으로 비활성화(`enabledByDefault: false`)되어 있으며, 관리자가 수동으로 스위치를 켜서 작동시켜야 합니다.

#### ③ Cleanup temporary workspaces (임시 워크스페이스 자동 청소)
*   **설명**: 개발자 API(`chat-sync`, `chat-async`)를 호출할 때 일시적으로 생성되는 `temp-` 접두사의 임시 워크스페이스들을 생성 후 24시간이 경과하면 자동으로 일괄 영구 삭제(내부 스레드, 대화 이력, 벡터 네임스페이스 포함)합니다.
*   **설정**: 시스템 작업 탭의 **"Auto-delete Temp Workspaces"** 설정 카드를 통해 전체 활성화/비활성화(ON/OFF)할 수 있습니다.
*   **특징**: API 연동을 통한 테스트나 일회성 채팅 시 발생하는 리소스 누수를 완벽하게 막아줍니다. (기본값: `Enabled` / 활성화)

## 2. API 호출 시 기존 워크스페이스 조회 및 재사용
ProjectM은 외부 API (`/api/v1/workspace/chat-sync`, `/api/v1/workspace/chat-async`)를 호출하여 대화를 수행할 때, 무조건 신규 임시 워크스페이스를 생성하지 않고 **기존에 생성된 일반 워크스페이스가 있는지 먼저 조회**한 뒤 재사용합니다.

*   **조회 흐름**:
    1.  전달된 `workspaceName`을 슬러그화(`Workspace.slugify`)하여 동일한 슬러그의 일반 워크스페이스를 조회합니다.
    2.  존재한다면, 해당 일반 워크스페이스 컨텍스트에서 RAG 및 대화를 진행하므로, 자동 청소(Cleanup) 대상에서 제외되어 대화 기록과 문서를 안전하게 영구 보존할 수 있습니다.
    3.  매칭되는 일반 워크스페이스가 없을 때에만 `temp-[슬러그]` 명칭으로 임시 워크스페이스를 생성하며, 이는 하루가 지난 후 백그라운드 스케줄러에 의해 자동으로 리소스를 반환하고 제거됩니다.

## 2. 텔레그램 봇 (Telegram Bot) 연동 가이드
모바일 텔레그램 앱에서 컴퓨터 문서를 탐색하고 AI RAG 질의응답을 수행하도록 연결하는 가이드입니다.

### Step 1: 텔레그램 봇 토큰 발급받기
1.  텔레그램 앱에서 `@BotFather`를 검색하여 대화를 시작합니다.
2.  `/newbot` 명령어를 전송합니다.
3.  봇의 이름(Name)과 사용자 이름(Username, `_bot`으로 끝나야 함)을 설정합니다.
4.  성공적으로 생성되면 HTTP API 토큰(`TELEGRAM_BOT_TOKEN`)을 복사해 둡니다.

### Step 2: 프로젝트 환경 변수 (.env) 설정
1.  `server/.env.development` (혹은 운영 환경의 `.env`) 파일을 엽니다.
2.  아래 환경 변수 항목을 찾거나 추가하고 발급받은 토큰 값을 기입합니다:
    ```env
    TELEGRAM_BOT_TOKEN="복사한_텔레그램_봇_토큰_값"
    ```
3.  백엔드 서버를 재부팅합니다. 서버 부팅 시 로그에 `[Telegram] Bot is starting...` 형태의 로그가 찍히며 자동으로 봇 클라이언트가 활성화됩니다.

### Step 3: 텔레그램에서 봇과 상호작용하기
1.  사용자 지정한 봇 주소(예: `https://t.me/MyProjectMBot`)로 들어가 **시작(/start)** 버튼을 누릅니다.
2.  **워크스페이스 연동**: 봇 메뉴 및 버튼 UI를 사용하여 질문할 대상 워크스페이스와 활성 대화 스레드를 골라 설정합니다.
3.  **대화 실행**: 일반 텍스트 질문을 전송하면, 봇이 해당 워크스페이스의 벡터 DB에서 지식을 인출(RAG)하여 신뢰도 높은 답변 메시지를 돌려줍니다.
